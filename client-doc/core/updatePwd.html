定期修改密码 ​

接口概述 ​

提供用户修改个人密码的安全接口，包含多重验证和强度检测机制。

接口定义 ​

java

@Log(title = "个人信息", businessType = BusinessType.UPDATE)
@PutMapping("/updatePwd")
@Operation(summary = "修改密码")
public AjaxResult updatePwd(@RequestBody Map<String, String> params)

参数说明 ​

参数名类型必填说明
oldPasswordString是当前使用的旧密码
newPasswordString是要设置的新密码

业务逻辑流程 ​

  1. 参数验证：提取并验证新旧密码参数
  2. 用户身份验证：获取当前登录用户信息
  3. 旧密码验证：检查旧密码是否正确
  4. 重复性检查：确保新密码不与旧密码相同
  5. 密码强度检测：调用PasswordStrengthChecker进行强度验证
  6. 密码加密：对新密码进行安全加密
  7. 数据更新：更新数据库中的用户密码
  8. 缓存同步：更新登录用户的缓存信息

密码强度检测器 ​

类定义 ​

PasswordStrengthChecker - 密码强度检测工具类

核心功能 ​

基础检测规则

  • 长度要求：8-20个字符
  • 字符类型要求：
    • 必须包含大写字母（A-Z）
    • 必须包含小写字母（a-z）
    • 必须包含数字（0-9）
    • 必须包含特殊字符（!@#$%^&*等）

安全限制规则

  • 禁止连续数字：防止"123"、"456"等连续数字序列
  • 禁止重复字符：防止"aaa"、"111"等连续相同字符
  • 禁止非法字符：只允许字母、数字和特定特殊字符

正则表达式模式 ​

java

// 检测连续数字的模式（3位及以上连续数字）
private static final Pattern SEQUENTIAL_DIGIT_PATTERN =
        Pattern.compile("(012|123|234|345|456|567|678|789|890|987|876|765|654|543|432|321|210)");

// 检测连续相同字符的模式
private static final Pattern REPEAT_CHAR_PATTERN =
        Pattern.compile("(.)\\1\\1");

// 特殊字符模式（常见的特殊字符）
private static final Pattern SPECIAL_CHAR_PATTERN =
        Pattern.compile("[!@#$%^&*()_+\\-=\\[\\]{};':\"\\\\|,.<>\\/?]");

主要方法 ​

1. checkPasswordStrength(String password) ​

功能：检测密码强度并返回结果

返回类型：PasswordCheckResult

  • isStrong：是否通过强度检测
  • message：检测结果信息
  • score：密码强度分数（0-10分）

2. 验证方法 ​

方法名功能说明
containsUpperCase()检查是否包含大写字母
containsLowerCase()检查是否包含小写字母
containsDigit()检查是否包含数字
containsSpecialChar()检查是否包含特殊字符
containsIllegalChars()检查是否包含非法字符
containsThreeSequentialDigits()检查是否包含连续数字

3. calculatePasswordScore(String password) ​

功能：计算密码强度分数

评分维度：

  • 长度得分：8-11字符(1分)、12-15字符(2分)、16+字符(3分)
  • 字符类型得分：包含每种字符类型各得1分（共4分）
  • 复杂度加分：
    • 特殊字符≥3个：2分
    • 特殊字符=2个：1分
    • 大小写字母比例均衡：1分
    • 数字字母交叉模式：1分

安全特性 ​

多重验证 ​

  • 旧密码验证：确保只有合法用户才能修改密码
  • 密码强度验证：防止设置弱密码
  • 重复性检查：防止新旧密码相同

加密保护 ​

  • 密码加密：使用SecurityUtils.encryptPassword()进行安全加密
  • 对比验证：使用SecurityUtils.matchesPassword()进行安全对比

缓存同步 ​

  • 用户信息更新：更新缓存中的用户密码信息
  • 时间戳更新：记录密码最后修改时间

日志记录 ​

  • 操作日志：记录密码修改操作
  • 业务日志：区分不同业务类型的操作

使用场景 ​

定期密码修改

用户按照安全策略定期修改密码。

安全事件响应

检测到异常登录行为后强制修改密码。

首次登录

新用户首次登录时强制修改初始密码。
